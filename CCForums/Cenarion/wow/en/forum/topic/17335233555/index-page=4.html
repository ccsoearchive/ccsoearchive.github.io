I have good news. It looks like Agent has been screwing up your permissions for you (yay). Why is that good news? Because I may have found a way to fix it on your own. It's a manual fix and involves using the Terminal app found in /Applications/Utilities, but it not only fixed the permissions problem I had (both of the errors found in this thread), but it also fixed SC2's "Whoops, something broke!" error and Hearthstone's "Updated to version 0.0.0.0" anomaly in the Battle.net app.

For the following instructions I'm going to assume you let the Battle.net app install your Blizzard games to the default directory, which is /Applications. If you installed to a different directory like I did so my games had their own SSD separate from the boot volume, you'll need to adjust for that in the Terminal command lines.

Warning: Pay special attention to capitalization in the command lines below. Capitalization matters when entering command arguments.

How to determine the current privileges for your WoW folder

1) Open the Terminal app found in /Applications/Utilities/.

2) Make a note of your account name in the command prompt. If you're using the default shell for the Terminal, it will be the short version of your full account name. So if your account name is Harry Potter, it should show something like this:

Fortress-Haximus:~ harrypotter$

You'll be using that "short" account name in the steps below to fix the problems found in this thread.

3) Type the following line into your Terminal window and then press Enter:

cd /Applications/World\ of\ Warcraft

4) Type the following line into your Terminal window and press Enter:

ls -l

You should end up with something like this:

Fortress-Haximus:World of Warcraft accountname$ ls -l
total 16
d---------+ 5 501 staff 170 Oct 14 2014 Cache
d---------+ 22 501 staff 748 Dec 16 11:18 Data
d---------+ 4 501 staff 136 Apr 18 2012 Interface
---x--x--x+ 1 501 staff 4 Apr 7 06:37 Launcher.db
d---------+ 13 501 staff 442 Dec 6 23:34 Logs
d---------+ 3 501 staff 102 Dec 14 21:44 Movies
d---------+ 33 501 staff 1122 Feb 9 00:02 Screenshots
d---------+ 5 501 staff 170 Jan 23 01:43 WTF
---x--x--x+ 1 501 staff 32 May 18 00:43 WoW.mfil
d---------+ 3 501 staff 102 Oct 14 2014 World of Warcraft Launcher.app
d---------+ 3 501 staff 102 Oct 14 2014 World of Warcraft.app

Notice how the only permissions here in my case are the "execute" (x) permissions. This indicates that the apps and files can only be executed, but not read or written to. Oops. Also in my case, the owner is the staff group. It should be the admin group. But we'll get to fixing that in just a bit. What you're looking for is to see if you have something like ---x--x--x+

Fixing your permissions

1) Open Terminal if not already open (it should still be open if you left it open from the above steps). You may wish to enlarge the window so you can see more lines at once.

2) Type the following line into your Terminal window:

sudo chown -R accountname:admin /Applications/World\ of \Warcraft

There will be a space after the folder name and then the cursor. Do not delete the space. Remember that accountname needs to be the short account name you see in the command prompt.

3) Press Enter.

4) You will be prompted to enter your admin password. Do so and press Enter.

5) Type the following line into your Terminal window and press Enter.

ls -l

6) You should then see a set of lines like this:

d---------+ 5 accountname admin 170 Oct 14 2014 Cache
d---------+ 22 accountname admin 748 Dec 16 11:18 Data
d---------+ 4 accountname admin 136 Apr 18 2012 Interface
---x--x--x+ 1 accountname admin 4 Apr 7 06:37 Launcher.db
d---------+ 13 accountname admin 442 Dec 6 23:34 Logs
d---------+ 3 accountname admin 102 Dec 14 21:44 Movies
d---------+ 33 accountname admin 1122 Feb 9 00:02 Screenshots
d---------+ 5 accountname admin 170 Jan 23 01:43 WTF
---x--x--x+ 1 accountname admin 32 May 18 00:43 WoW.mfil
d---------+ 3 accountname admin 102 Oct 14 2014 World of Warcraft Launcher.app
d---------+ 3 accountname admin 102 Oct 14 2014 World of Warcraft.app

Steps #1-#7 changed the owner of the World of Warcraft folder to your current account and the admin group.

Now we're going to actually fix the permissions themselves. We have to do it the hard way because the finder's Get Info box isn't working right, otherwise we'd have beaten this already. So onto the next set of steps to finish this up.

8) Type the following line into your Terminal window and press Enter:

sudo chmod -R 777 /Applications/World\ of\ Warcraft

8b) If you are prompted to enter your administrator password, do so and press Enter at this time.

Your permissions are now set to the maximum allowed (basically everything but root permissions) for the World of Warcraft folder.

9) Type the following line into your Terminal window and press Enter:

cd /Applications/World\ of \Warcraft

10) Type the following line into your Terminal window and press Enter:

ls -l

You should see a list like this:

Fortress-Haximus:World of Warcraft accountname$ ls -l
total 16
drwxrwxrwx+ 5 accountname admin 170 Oct 14 2014 Cache
drwxrwxrwx+ 22 accountname admin 748 Dec 16 11:18 Data
drwxrwxrwx+ 4 accountname admin 136 Apr 18 2012 Interface
-rwxrwxrwx+ 1 accountname admin 4 May 21 02:09 Launcher.db
drwxrwxrwx+ 13 accountname admin 442 Dec 6 23:34 Logs
drwxrwxrwx+ 3 accountname admin 102 Dec 14 21:44 Movies
drwxrwxrwx+ 33 accountname admin 1122 Feb 9 00:02 Screenshots
drwxrwxrwx+ 5 accountname admin 170 Jan 23 01:43 WTF
-rwxrwxrwx+ 1 accountname admin 32 May 18 00:43 WoW.mfil
drwxrwxrwx+ 3 accountname admin 102 Oct 14 2014 World of Warcraft Launcher.app
drwxrwxrwx+ 3 accountname admin 102 Oct 14 2014 World of Warcraft.app

If the permissions on the far left of your list match what shows here, you've successfully set your permissions for the WoW folder.

At this point you can close Terminal and run the Battle.net App. It should allow you to login and update properly. You'll know you've succeeded in fixing the permissions issue if you see the Play button (do not click that button if there is a progress bar on the screen though!). You should also at this point in time see a [v] button next to the Play button indicating you can download the pre-patch data if you have not done so already.

If any of your other games are affected by this bug, you will need to perform the steps above, substituting those game folders instead of the World of Warcraft folder. Just remember that folders with more than one word in the name use a forward slash "\" and a space after each word in the name except for the last word. Alternatively you could enter the folder name like this:

/Applications/"World of Warcraft"

or

/Applications/"StarCraft II"

Putting your folder name into quotes means you don't have to use escape characters and it's much more readable. But the choice is up to you.

For those wondering why I chose 777 as the permissions instead of 775, there have been instances where Agent could not properly elevate itself to admin status with only 775 permissions, so 777 has been the fix for that when using the Terminal (you can't set 777 permissions in the Finder's Get Info window, which is why it isn't working since currently Agent is not being given permission to be made the frontmost app by SecurityAgent (an OS X security daemon). Under most circumstances 775 permissions are desired, but in this case you need 777.
