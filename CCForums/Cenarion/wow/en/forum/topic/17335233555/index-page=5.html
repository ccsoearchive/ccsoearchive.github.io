I have been able to successfully launch the app with the following sequence:

1. Leave the Battle.net app running
2. If it says "Update" then click update and let the process finish.
3. Once the Button changes to "Play" do the following:
3.a. Open up the folder containing the "World of Warcraft" folder (~/Applications by default)
3.b. "Get Info" (Right click - Get Info OR highlight the folder and do a "Command+I") on the "World of Warcraft" folder
3.c. Expand "Sharing and Permissions"
3.d. Click the padlock and authenticate as an admin
3.e. Make sure each "Name" listed has "Read and Write" privilege, change each one of not
3.f. Click the Cogwheel and select "Apply to enclosed items"
3.g. Click Ok and wait for the process to complete
4. Switch back to Battle.net and click Play

This has worked 100% of the time since the issue appeared.

You can break it again by applying the update and then clicking play without repairing the permissions.

Although many have stated this doesn't appear to be a permissions problem, I've just helped a friend who has BLZBNTAGT00000BB8 and it was fixed by permissions.

These are the steps I asked him to take:

1. Open Terminal (command + space + type Terminal, press Enter)
2. On command prompt type the following (after the $ sign):
$ id
id=501(yourusername) gid=20(staff) groups=20(staff),702(com.apple.sharepoint.group.2),12(everyone),61(localaccounts),79(_appserverusr),80(admin),81(_appserveradm),98(_lpadmin),703(com.apple.sharepoint.group.4),701(com.apple.sharepoint.group.1),33(_appstore),100(_lpoperator),204(_developer),398(com.apple.access_screensharing),399(com.apple.access_ssh),401(com.apple.sharepoint.group.3)

The above should roughly be similar for everyone. But basically you should note 3 things:
501(yourusername) - where yourusername is YOUR username being bob, mary, joe - whatever - but it should certainly be 501
gid=20(staff) - yep, same for everyone
80(admin) - This means you're logged in with an admin account

Now moving forward.... as long as the above three are true

3. Change to the Applications Directory
$ cd /Applications/
4. List the permissions as they currently stand:
$ ls -l | grep World
drwxr-xr-x 22 username staff 748 29 May 20:55 World of Warcraft

The above is normal. You should see:
drwxr-xr-x - which means:
d = directory
1st 3 - rwx = User (username) can read/write/execute (all directories must execute to enter)
2nd 3 - r-x = Group (staff) can read/execute
3rd 3 - r-x = World can read/execute

Now what my friend had was incorrect:
drwxr-xr-x 22 root admin 748 29 May 20:55 World of Warcraft
Which meant the user could do bugger all.

4. Fix permissions - anyone can run this safely (as long as you type the whole thing):
$ sudo chown -R 501:20 World\ of\ Warcraft/
<ENTER YOUR PASSWORD and wait

5. Once complete try running Battle.net again

If it's still broken....

6. Move directory to a new name temporarily
$ mv World\ of\ Warcraft/ World\ of\ Warcraft2/

7. Re-run Battle.net and you will be prompted to install World of Warcraft
8 Start installing World of Warcraft, cancel after the indicies
9. Quit Battle.net app
10. Back in Termal run the following:
$ mv World\ of\ Warcraft/ World\ of\ Warcraft_new/
$ mv World\ of\ Warcraft2/ World\ of\ Warcraft/
11. Re-run Battle.net and the Run button should be gray and the application should be upgrading without any prompt
