Why does it matter which comes first? I assume the attack table works like this.. roughly

Say you have:

5% miss -> 5% unhittable
12% dodge -> 17% unhittable
14% parry -> 31% unhittable
40% block -> 71% unhittable.

I'd assume the game effectively runs a random number as the swing is thrown (against an equal level mob with no special buffs)

If it's 1-5, the attack misses
If it's 6-17, the attack is dodged
if it's 18-31, the attack is parried
if it's 32-71, the attack lands but is blocked
If it's 72->100, the attack lands

If parry comes before dodge, it changes what range of numbers triggers a parry, but it doesn't change how often it's likely to happen.

If a mob was 85 but had +2% hit, then it would be (also changing for the order of dodge and parry, just for demonstration)

If it's 1-3, the attack misses
If it's 4-17, the attack is parried
if it's 18-29, the attack is dodged
if it's 30-69, the attack lands but is blocked
If it's 70->100, the attack lands
